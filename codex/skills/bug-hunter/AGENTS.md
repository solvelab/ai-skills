# Bug-Hunter Skill

Adversarial testing rite: try to break the change, don't just confirm the happy path. Two tracks —
Backend (pytest: anti-forge, clamps, atomicity, dependency-down, concurrency) and FiveM/Lua (pure
modules, event injection, NUI payload, disconnect cleanup, StateBag races, fallback under failure).

## When to use

Writing/reviewing tests for a change; a tasks.md "Testes & Bug-Hunter" group; user says bug hunt,
adversarial test, edge cases, atomicity, anti-forge, race condition.
